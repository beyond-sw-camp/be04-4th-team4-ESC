# 1. 프로젝트 개요
|영어의 영향력|취업시장에서의 필요성|
|---|---|
| ![1b6120e63ede4fa8c512961ace77c03f](https://github.com/dongh810/ESC-English_Study_Club-develop/assets/105986200/7c67397e-0f68-4bb1-b688-0d59c3c8c8a2) |![34055_22894_2547](https://github.com/dongh810/ESC-English_Study_Club-develop/assets/105986200/a8493f15-64d7-48db-ae7c-35cfeb552450)|

2024년 현재 영어는 앞으로 살아감에 있어서 빠질수가 없는, 필수로 익혀야 하는 언어입니다. 취업을 위한 스펙으로 영어 자격증이나 영어 회화능력은 더이상 특별한것이 아닌 기본적인 것으로 자리잡았고, 외국에 나가서 소통을 하거나, 개발공부를 위해 구글링을 할때에도 영어로 되어있는 자료들이 많아 영어의 필요성은 모두 느끼고 있을 것 입니다.

ESC(English Study Club) 프로젝트는 영어의 필요성을 깨닫고, 혼자 영어 공부할 때 집중력을 유지하기 어렵다고 느끼거나 그룹 스터디를 하고싶은 사람들을 위해서 만들어졌습니다. 영어자격증 시험을 목표로 스터디원을 모으고 사용자들이 서로의 시간을 공유하여 함께 공부할 수 있는 최적의 시간을 찾아줌으로써 스터디 시간을 효율적으로 활용할 수 있도록 도와줄 것 입니다.

## 주요기능
- 스터디 클럽 생성 후 스터디원 모집
- 채팅을 사용하여 모집하는 스터디에 대해 궁금증 해소 가능
- 개인 일정 등록과 스터디 일정 등록
- 스터디 후 개인 스터디로그 작성

## 새롭게 도입한 기술
- FullCalendar: 일정 관리 (FullCalendar API)
- Websocket: 회원 간 채팅 기능 구현 (spring-boot-starter-websocket,  Vanilla Javascript Websocket)  
- Crawling: 토익, 오픽 등 실제 시험일정 조회 (JSoup)
- SMTP(Simple Mail Transfer Protocol): 이메일 인증을 통한 회원관리
- Jasypt: 설정 파일 및 민감한 정보 암호화 (Jasypt)
- Monitoring: Prometheus, Grafana를 사용한 백엔드 모니터링

# 3. 고도화 계획

## 로그인 기능
- 사용자 인가를 통해 권한별로 접근 가능한 요청들을 분리해서 회원들을 관리하도록 구현

## 일정 기능
- 일정을 합칠 때 여러 개 합칠 수 있게 구현
- 겹치는 일정 조회 시 겹치는 시간이 많을수록 진하게 표현

## 채팅 기능 
- 1:1 채팅방 구현
- 채팅 메세지 백엔드 처리
- 채팅방 입장/퇴장 메세지 추가
- Vite <-> SpringBoot 간 STOMP 라이브러리를 사용할 수 있는 방법 찾기

## 크롤링 기능
- 크롤링하는 사이트에 새로 등록되는 시험일도 추가될 수 있게 주기적으로 자동 실행되도록 구현

## 모니터링 
- Springboot 이외에 DB server, Vue Project, Jenkins 등 등록된 다른 서비스 모니터링 추가 

## CICD
- Jenkins에서 Dockerhub image upload 후 자동으로 Kubernetes 리롤 할 수 있도록 인프라 공부해보기 (완전 자동화 구현)

| 정우진 | 이번 2주 동안 진행한 프로젝트는 기존에 습득한 지식을 기반으로 DDD 설계와 모델링을 통해 데이터베이스를 설계하고 구축했습니다. 그 후 백엔드와 프론트엔드를 모두 구현하고, Docker, Kubernetes, Jenkins를 활용하여 CI/CD를 통해 버전 관리와 배포를 경험했습니다. 이번 프로젝트에서는 각자 처음 접하는 새로운 기술이나 라이브러리를 도입하는 것을 목표로 하였습니다. 특히, ModelMapper 대신 MapStruct 라이브러리를 적용해 보았는데, ModelMapper의 경우 Entity에 setter를 필요로 하기에 그보다 효율적인 MapStruct를 선택하게 되었습니다. Jenkins를 활용하여 파이프라인을 구축한 후 코드 변경 시 테스트 코드를 기반으로 테스트하는 과정을 경험하면서 TDD의 중요성을 알게 되었습니다. 이러한 경험을 통해 프로젝트 전체적인 흐름을 파악하고, 개발 과정에서의 안정성과 효율성을 높일 수 있었습니다. <br> 끝으로, 이번 프로젝트는 새로 합류한 팀원들과 함께 진행되었으며, 팀원들과의 소통과 협력을 통해 문제를 해결하고 발전하는 과정을 경험했습니다. 최종 프로젝트에서는 팀원들의 다양한 의견을 수용하고, 합의점을 찾아가며 더 나은 결과물을 만들기 위해 노력할 것입니다. 함께 더 큰 성과를 이루어내기 위해 노력하겠습니다. |
